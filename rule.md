# Slay the Balatro 游戏规则

## 游戏简介
Slay the Balatro 是一款卡牌策略游戏，玩家通过抽牌、打牌来累积分数。游戏中包含三种类型的卡牌，每种卡牌都有独特的效果。玩家需要合理利用卡牌的特性和组合，最大化自己的得分。**注意，你只需要实现游戏的服务端，即模拟玩家的操作（输入）和游戏逻辑。**

## 游戏组件

### 1. 牌堆系统
游戏中存在三个独立的牌堆：

- **抽牌堆（Draw Pile）**：用于抽取卡牌的牌堆，初始时包含所有卡牌
- **手牌堆（Hand）**：玩家当前持有的卡牌，可以选择打出
- **弃牌堆（Discard Pile）**：已经打出的得分牌和技能牌会进入弃牌堆

#### 牌堆的数据结构说明

所有牌堆都使用**链表**实现，每个牌堆都有明确的**头部（head）**和**尾部（tail）**，我们后面会一直使用`head`和`tail`来表示牌堆的两端：

```
head (第1张) → 第2张 → 第3张 → ... → tail (最后一张)
```

### 2. 卡牌类型
游戏中有三种卡牌类型：

#### 得分牌（Score Card）
- **标识**：`SCORE`
- **属性**：包含一个分数值（point）
- **效果**：打出时根据当前游戏状态获得相应分数
  - 最终得分 = (卡牌分数值 + 当前累积的能力加成) × 当前倍率
  - 打出后，倍率自动重置为 1
- **打出后**：进入弃牌堆（塞入弃牌堆尾部）

#### 技能牌（Skill Card）
有两种技能类型：

##### 1) Multiplier（倍率增加）
- **标识**：`SKILL Multiplier`
- **效果**：增加下一张得分牌的倍率
  - 每使用一次，下一张得分牌的倍率增加 1
  - 只对下一张得分牌生效，该得分牌使用后倍率重置为 1
  - 可以叠加使用：连续打出多张 Multiplier 可以累积倍率
  - 最终得分 = (卡牌基础分数 + 能力加成) × 当前倍率
- **打出后**：进入弃牌堆（塞入弃牌堆尾部）

##### 2) HeadButt（头槌）
- **标识**：`SKILL HeadButt`
- **效果**：将弃牌堆的 **tail（最底端/最后一张）** 移动到抽牌堆的 **head（最顶端/第一张）**
  - 如果弃牌堆为空，则无效果
  - 该牌会成为下一次从抽牌堆抽取的第一张牌
- **打出后**：进入弃牌堆（塞入弃牌堆尾部）

#### 能力牌（Power Card）
- **标识**：`POWER`
- **属性**：包含一个加成值（plus_count）
- **效果**：永久增加得分加成
  - 加成值会累积到游戏的能力加成状态中
  - 之后每张得分牌打出时，都会额外获得这个累积的加成值
  - 加成是永久性的，会一直保持到游戏结束
- **打出后**：卡牌被销毁（不进入弃牌堆）

## 游戏机制

### 1. 初始化
- 游戏开始时，玩家选择一种洗牌模式（1、2 或 3）
- 所有卡牌被加载到抽牌堆中
- 初始分数为 0，无任何状态效果，倍率为 1

### 2. 抽牌机制（draw 命令）
- 每次抽牌尝试抽取 **5 张卡牌**
- **详细流程**：
  1. 从抽牌堆的 **head（顶部）** 移除一张牌
  2. 将这张牌添加到手牌堆的 **tail（尾部）**
  3. 重复 5 次
- **特殊情况**：
  - **在抽牌的过程中**如果抽牌堆为空，会触发洗牌机制（将弃牌堆的牌洗回抽牌堆）。注意，**并不是抽牌堆为空**就会触发洗牌机制，而是**在抽牌过程中**发现抽牌堆为空时，**且弃牌堆不为空**，才会触发洗牌机制。比如说你要抽 5 张牌，但抽到第 3 张时发现抽牌堆空了，且弃牌堆有牌，那么就会先洗牌，然后继续抽剩下的牌。而如果你要抽 5 张牌，抽牌堆有5张牌，你抽完5张牌后变空了，这种情况就不会触发洗牌机制，必须等到下一次抽牌时才会触发。
  - 抽牌堆和弃牌堆的牌数可能小于 5 张。即使未满 5 张也不会报错，**能抽多少抽多少**

**抽牌示例**：
```
抽牌前：
  抽牌堆: [A, B, C, D, E]  (A 是 head)
  手牌堆: [X, Y]           (X 是 head, Y 是 tail)

抽取 2 张后：
  抽牌堆: [C, D, E]        (C 是新的 head)
  手牌堆: [X, Y, A, B]     (X 是 head, B 是新的 tail)
```

### 3. 打牌机制（play 命令）
- 命令格式：`play <card_index>`
- `card_index` 是手牌的位置索引（**从 1 开始计数，从 head 开始数**）
- 执行流程：
  1. 从手牌堆的 **head 开始**，找到第 `card_index` 张牌并移除
  2. 执行该卡牌的效果
  3. 如果是得分牌或技能牌，将其加入弃牌堆尾部
  4. 如果是能力牌，直接销毁该卡牌

### 4. 洗牌机制
有三种洗牌模式：

#### 模式 1：outShuffle（顺序洗牌）
- 将整个弃牌堆直接追加到抽牌堆的 **tail（底部）**
- 顺序保持不变
- **示例**：
  ```
  洗牌前：
    抽牌堆:  []
    弃牌堆:  [A, B, C, D]  (A 是 head, D 是 tail)
  
  洗牌后：
    抽牌堆:  [A, B, C, D]
    弃牌堆:  []
  ```

#### 模式 2：inShuffle（逆序洗牌）
- 将弃牌堆逆序塞入抽牌堆的 **tail（底部）**
- **示例**：
  ```
  洗牌前：
    抽牌堆:  []
    弃牌堆:  [A, B, C, D]  (A 是 head, D 是 tail)
  
  洗牌后：
    抽牌堆:  [D, C, B, A]
    弃牌堆:  []
  
  下次抽牌顺序: D, C, B, A
  ```

#### 模式 3：oddEvenShuffle（奇偶洗牌）
- 将弃牌堆的牌按索引位置（从head开始数，head所对应卡牌索引为1）分为奇数和偶数两组
  - 索引 1，3，5，7... 的牌先追加到抽牌堆
  - 索引 2，4，6，8... 的牌后追加到抽牌堆
- 各组内部相对顺序保持不变
- **示例**：
  ```
  洗牌前：
    抽牌堆:  []
    弃牌堆:  [A, B, C, D, E]  (A 是 head, 索引 1)
  
  分组：
    奇数索引组: [A(1), C(3), E(5)]
    偶数索引组: [B(2), D(4)]

  洗牌后：
    抽牌堆:  [A, C, E, B, D]
    弃牌堆:  []
  
  下次抽牌顺序: A, C, E, B, D
  ```

### 5. 结束回合（finish 命令）
- 将所有剩余手牌**按顺序**移动到弃牌堆的 **tail（底部）**

**示例**：
```
结束前：
  手牌堆: [A, B, C]
  弃牌堆: [X, Y]
  
结束后：
  手牌堆: []
  弃牌堆: [X, Y, A, B, C]
```

## 游戏命令

### 操作命令

| 命令 | 格式 | 说明 |
|------|------|------|
| draw | `draw` | 抽取 5 张卡牌到手牌 |
| play | `play <card_index>` | 打出手牌中第 card_index 张牌（从 1 开始） |
| finish | `finish` | 结束回合，将手牌移至弃牌堆 |
| exit | `exit` | 退出游戏 |

### 查询命令

| 命令 | 格式 | 返回值 |
|------|------|--------|
| query score | `query score` | 当前总分数 |
| query hand | `query hand` | 手牌堆中的牌数 |
| query draw_pile | `query draw_pile` | 抽牌堆中的牌数 |
| query discard_pile | `query discard_pile` | 弃牌堆中的牌数 |

## 分数计算详解

得分牌的分数计算遵循以下公式：

```
最终得分 = (卡牌基础分数 + 累积能力加成) × 当前倍率
```

其中：
- **卡牌基础分数**：得分牌自身的 point 值
- **累积能力加成**：所有已打出的能力牌的 plus_count 之和
- **当前倍率**：初始为 1，每使用一次 Multiplier 技能牌就增加 1，使用得分牌后重置为 1

### 计算示例

假设游戏进行如下操作：

1. 打出 `POWER 5`（能力牌，加成 +5）
   - 累积能力加成：0 + 5 = 5
   - 当前倍率：1
   - 当前总分：0

2. 打出 `SCORE 10`（得分牌，分数 10）
   - 实际得分：(10 + 5) × 1 = 15
   - 倍率重置为 1
   - 当前总分：0 + 15 = 15

3. 打出 `SKILL Multiplier`
   - 当前倍率：1 + 1 = 2
   - 当前总分：15

4. 打出 `POWER 3`（能力牌，加成 +3）
   - 累积能力加成：5 + 3 = 8
   - 当前倍率：2（保持不变）
   - 当前总分：15

5. 打出 `SCORE 7`（得分牌，分数 7）
   - 实际得分：(7 + 8) × 2 = 30
   - 倍率重置为 1
   - 当前总分：15 + 30 = 45

6. 打出 `SKILL Multiplier`（第一次）
   - 当前倍率：1 + 1 = 2
   - 当前总分：45

7. 打出 `SKILL Multiplier`（第二次）
   - 当前倍率：2 + 1 = 3
   - 当前总分：45

8. 打出 `SCORE 5`（得分牌，分数 5）
   - 实际得分：(5 + 8) × 3 = 39
   - 倍率重置为 1
   - 当前总分：45 + 39 = 84

## 输入格式

### 初始输入
```
<shuffle_mode>
<n>
<card_type_1> [参数]
<card_type_2> [参数]
...
<card_type_n> [参数]
```

- `shuffle_mode`：洗牌模式（1/2/3）
- `n`：卡牌总数
- 每行一张卡牌：
  - 得分牌：`SCORE <point>`
  - 技能牌：`SKILL <Multiplier|HeadButt>`
  - 能力牌：`POWER <plus_count>`

### 命令输入
在初始输入后的输入都是上面所叙述的操作命令和查询命令。

## 注意事项

- 所有索引在命令中都是从 1 开始计数（如 `play 1` 表示打出第一张手牌）,其中的计数都是从 head 开始数
- 能力牌的加成会一直保持，直到游戏结束
- Multiplier 只对下一张得分牌生效，使用得分牌后倍率立即重置为 1
- 多次使用 Multiplier 效果**会叠加**，可以累积倍率（例如：使用 3 次后，倍率变为 4）
- 当抽牌堆和弃牌堆都为空时，无法继续抽牌
- HeadButt 对空弃牌堆使用时无效果
